Implement a simple receiving module (Recepción simple) in FastWMS.

GOAL
Add a practical inbound receiving workflow to FastWMS for day-to-day warehouse operation, without replacing the existing bulk stock upload process.

IMPORTANT
This module is complementary to stock upload.
- Bulk stock upload remains valid for initial loads, syncs, and administrative corrections.
- Receiving is for operational inbound activity:
   purchases, returns, transfers, adjustments, restocking, etc.

This is a simple receiving module for V1, not a full enterprise ASN/PO receiving system.

==================================================
PART 1 — BUSINESS SCOPE
==================================================

We need a basic receiving workflow so warehouse staff can:
- create a receiving run
- register received SKUs and quantities
- assign destination location
- assign stock bucket (MAIN / WEB)
- confirm receipt
- increase WMS inventory with audit trail

Supported V1 scope:
- manual receiving
- bulk file receiving (optional but recommended)
- destination location required
- stock bucket required
- mobile-friendly execution optional for later
- no ASN/PO matching yet
- no lot/serial control yet

==================================================
PART 5 — RECEIVING CONFIRMATION LOGIC
==================================================

Create service:
confirm_receiving_run(run_id)

Behavior:
1) For each WmsReceivingLine:
   - resolve/create physical WmsLocation
   - resolve/create Product if needed
   - find or create WmsInventory row by:
      location_id + product_id + stock_bucket
   - increment WmsInventory.on_hand_units by qty_received
   - do not mix buckets
   - do not touch allocated_units unnecessarily

2) Mark line status = RECEIVED
3) Mark run status = COMPLETED
4) Write receiving audit log or movement log

Important:
- If same physical location already has same SKU in same bucket, accumulate units
- If same physical location has same SKU in different bucket, keep separate WmsInventory rows

==================================================
PART 6 — LOCATION / STOCK BUCKET RULES
==================================================

Receiving must respect the existing shared-location model:
- physical locations are shared
- stock is separated only by stock_bucket

Examples:
Location A-02 / Box 01 / SKU X
- receiving 5 units into MAIN increases only MAIN bucket
- receiving 5 units into WEB increases only WEB bucket

Never merge MAIN and WEB quantities together.

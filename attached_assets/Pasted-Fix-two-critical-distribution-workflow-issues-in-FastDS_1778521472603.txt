Fix two critical distribution workflow issues in FastDSO:

1) distribution edits are not being persisted correctly or are not reflected in exports
2) the distribution edit screen does not implement a real per-SKU editable allocation pool when units are removed from one store and reassigned to others

GOAL
Make the distribution review/edit flow truly operational:
- edits must persist reliably in qty_final
- exports must use the saved final quantities
- users must be able to manually reallocate freed units across stores through a per-SKU pool concept
- distribution history/export should move out of the main dashboard into a dedicated module

- not being saved to the database
- or not being reflected when exporting distributions from the main dashboard

- show success message

Audit all export logic related to distributions.

- qty_suggested

3) Add debug logs:

[PLAN] export using qty_final plan_id=...

PART 3 — PER-SKU EDITABLE POOL LOGIC
==================================================

PROBLEM
When the user reduces qty_final for one store, those freed units do not become available as a reusable pool for the same SKU.

GOAL
For each SKU, the edit screen must maintain a real live allocation pool so the user can manually reassign freed units to any other store(s).

CORE RULE

For each SKU:
pool_sku = available_main_for_sku - sum(qty_final across all stores for that SKU)

Where:
- available_main_for_sku = current MAIN CD available stock for that SKU
- WEB stock must never be used here

REQUIRED BEHAVIOR

1) If the user decreases qty_final in one store:
- pool_sku must increase immediately

2) If the user increases qty_final in another store:
- pool_sku must decrease immediately

3) The user must be able to assign the freed units to:
- one other store
- or multiple stores
manually

4) If user tries to increase beyond pool_sku:
- show immediate warning
- optionally block that increase

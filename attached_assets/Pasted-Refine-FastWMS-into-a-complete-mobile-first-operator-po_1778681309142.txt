Refine FastWMS into a complete mobile-first operator portal for daily warehouse execution.

GOAL
Build a truly usable operator mobile experience inside FastWMS.
The current mobile portal must evolve from “it works” to “it is operationally comfortable and coherent”.

SCOPE
This prompt focuses on the operator experience for:
1) My Waves
2) Picking execution
3) Picking issues
4) Moves between locations
5) Inventory by location

Do NOT change desktop WMS behavior except where needed for shared logic.
All /wms/mobile/* routes must use dedicated mobile templates and mobile-first layout.

==================================================
PART 1 — MOBILE DESIGN FOUNDATION
==================================================

1) Create/finish a dedicated mobile layout:
- templates/mobile_base.html

Requirements:
- proper viewport meta
- no desktop sidebar
- compact top header with title/logo
- optional bottom nav for operator tasks
- mobile spacing, cards, badges, large tap targets
- support sticky bottom action bars
- keep FastDSO/FastWMS visual style

2) Operator mobile navigation
Provide a simple navigation structure with:
- Mis Waves
- Picking
- Incidencias
- Movimientos
- Inventario
- Salir

Navigation must feel native/mobile, not like desktop links stacked randomly.

- MANUAL_WEB
- MANUAL_OTHER

3) Card content per wave:

   PLANNER
   WEB

   MAIN / WEB
- Origen físico if useful
- Urgencia
- Estado

   Iniciar / Continuar

PART 3 — PICKING EXECUTION SCREEN
==================================================

Route:
- /wms/mobile/wave/<wave_id>

Requirements:
1) Show one active reservation/task at a time
Do not dump large tables.

2) Main card must show clearly:
- Ubicación
- Caja
- Código ubicación
- Bucket de stock
- SKU
- Producto
- Cantidad requerida
- Cantidad pickeada
- Cantidad restante
- Estado de la tarea/reserva

3) Mandatory scan flow
Location scan is required.
SKU scan must be validated before allowing pick actions.

Show scan sections with:
- Escanear ubicación
- Escanear SKU
- manual fallback input if camera unavailable

4) Big action buttons
Sticky bottom action area with:
- Pick completo
- Pick parcial
- Reportar incidencia

5) Auto-advance
After successful pick or issue flow:
- move to next reservation/task automatically
- show short success feedback

6) Reallocated reservations
If a reservation is reallocated:
- show badge:
   Reasignada
- still display real physical location
- keep shared-location / stock-bucket context
